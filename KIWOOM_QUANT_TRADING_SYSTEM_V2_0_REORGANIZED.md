# Kiwoom Quant Trading System v2.0（重整版）

## 一、系统定位

Kiwoom Quant Trading System v2.0 不是单一策略，而是面向韩国股市 REAL ONLY 实盘交易的统一主控系统。

系统统一管理：
- 市场时段判断
- 策略窗口路由
- 执行闸门
- 风险控制
- 主量化策略
- 超跌修复策略
- 订单执行与成交跟踪
- 次日卖出与盘中退出纪律

核心原则：
- REAL ONLY
- Asia/Seoul 为唯一交易时钟
- 三层永久分离：整体市场判断 / 候选池判断 / 新闻风险判断

---

## 二、市场时段框架

- 08:30 - 09:00 → PRE_MARKET
- 09:00 - 09:05 → OPENING_CONFIRMATION
- 09:05 - 09:25 → MORNING_EXECUTION
- 09:25 - 14:50 → INTRADAY_MANAGEMENT
- 14:50 - 15:20 → OVERSOLD_SCAN
- 15:20 - 15:28 → PRE_CLOSE_EXECUTION
- 15:28 - 15:30 → PRE_CLOSE_AUCTION
- 15:40 - 16:00 → POST_CLOSE
- 16:00 - 18:00 → AFTER_HOURS
- 其他时间 → CLOSED

## 四、超跌修复模型（最新固化版）

### 1. 股票池限制
只看：
- KOSPI200
- KOSDAQ150

禁止：
- 扩展到其他普通 KOSPI / KOSDAQ 股票池
- 因为“跌得多”就把非成分股混进来

### 2. 目标
寻找当天尾盘出现超跌、但仍具修复可能的 KOSPI200 / KOSDAQ150 成分股，尾盘买入，隔夜持有，次日主动卖出。

### 3. 三层分析结构
#### A. 整体市场层
看：
- KOSPI 走势
- KOSDAQ 走势
- 主板 / 创业板强弱对比
- 市场是否 risk-off

#### B. 候选池层
每只候选股默认展示：
- 代码
- 名称
- 所属市场（KOSPI200 / KOSDAQ150）
- 前一天收盘价
- 现价
- 日内最高
- 日内最低
- 当日跌幅
- 修复强弱

#### C. 新闻风险层
每次推荐必须同时检查：
- 市场新闻风险
- 行业新闻风险
- 个股明显利空

查不到就直接写：查不到

### 4. 跌幅区间
- 优先：-4% ~ -7%
- 次选：-7% ~ -10%
- 跌超 -10%：默认高风险观察，不优先

### 5. 尾盘修复确认
进入执行池前重点确认：
- 当前价高于日内最低点一定幅度
- 14:30 后跌幅开始收窄
- 尾盘不再持续创新低
- 承接改善
- 不是一路被砸到收盘

15:00 后仍持续创新低的票，默认降级或剔除。

### 6. 买入时间
- 15:00 - 15:20：观察、筛选、确认
- 15:20 - 15:28：正式执行
- 15:28 后原则上不再新增仓位

### 7. 次日卖出规则（最新）
- 涨幅达到 +5%：先卖出 50%
- 跌破 -2%：立即止损
- 次日主板 / 创业板实时走弱：可直接卖出
- 主要卖出时间参考：提前到次日 10:00

### 8. 订单管理纪律
- 普通订单 1 分钟未成交：主动查状态、撤单/改单、重新下单
- 交易成功后：立即主动通知用户结果
- 关键时间点（如 15:00、15:20、次日 10:00 前）：必须主动推送，不等用户追问

## 五、标准输出结构

以后候选推荐统一按以下结构输出：

1. 市场层
   - KOSPI 走势
   - KOSDAQ 走势
   - 主板 / 创业板强弱
   - 风险结论

2. 候选池
   - 代码 / 名称
   - KOSPI200 / KOSDAQ150
   - 前收 / 现价 / 最高 / 最低
   - 当日跌幅
   - 修复强弱

3. 新闻风险层
   - 市场新闻风险
   - 行业新闻风险
   - 个股明显利空 / 查不到

4. 模型判断
   - 为什么入选
   - 为什么只是观察
   - 为什么可执行 / 不可执行

5. 最终结论
   - 做 / 不做 / 观望

## 六、执行纪律

- 数据不完整，不强行下结论
- 查不到就直接说查不到
- 不用旧缓存冒充实时结果
- 关键时间点默认主动执行与主动汇报
- 超跌修复模型只看 KOSPI200 / KOSDAQ150，不再扩展范围
