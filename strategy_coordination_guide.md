└── 现金储备: 10%

## ⚖️ **冲突解决优先级规则**

### **规则1：时间敏感度优先**
```
超跌反弹模型 > 主要交易模型
```
**理由**: 超跌机会窗口短（09:05-09:25买入，13:00前卖出）

### **规则2：资金使用优先级**
```
1. 超跌模型日内循环资金 (≤20%)
2. 主要模型长期占用资金 (≤70%)
3. 现金储备 (≥10%)
```

### **规则3：股票持仓冲突**
```
if 股票在主要模型持仓中:
    if 超跌模型信号强烈:
        允许超跌模型买入 (短线覆盖长线)
    else:
        禁止超跌模型买入
else:
    按各自模型规则执行
```

## 🚨 **风险控制层级**

### **第一层：单日总亏损限制**
```
if 当日总亏损 ≥ 总资金2%:
    暂停所有新开仓
    只允许平仓操作
```

### **第二层：策略级亏损限制**
```
if 超跌模型连续2笔亏损:
    当天暂停超跌模型
    主要模型继续运行

if 主要模型单日亏损 ≥ 该策略资金5%:
    主要模型减仓至50%
```

### **第三层：市场环境控制**
```
market_score 规则:
- ≥1: 正常交易
- 0: 轻仓交易 (各策略减半)
- ≤-1: 暂停超跌模型，主要模型减仓
```

## 🔧 **具体操作流程**

### **每日盘前 (08:50)**
```python
# 1. 检查市场环境
market_score = calculate_market_score(
    us_change=美股涨跌,
    kr_preopen=韩股预期,
    vix_change=波动率变化
)

# 2. 确定当日策略
if market_score >= 1:
    # 正常模式
    oversold_allocation = 0.20  # 20%
    main_allocation = 0.70      # 70%
elif market_score == 0:
    # 谨慎模式
    oversold_allocation = 0.10  # 10%
    main_allocation = 0.60      # 60%
else:
    # 防御模式
    oversold_allocation = 0.00  # 0%
    main_allocation = 0.50      # 50%
```

### **盘中交易 (09:00-15:00)**
```python
# 超跌模型执行窗口: 09:05-09:25
# 主要模型执行: 全天，但避免与超跌模型冲突

def execute_trade(strategy_name, trade_signal):
    # 检查冲突
    allowed, reason = coordinator.check_conflicts(strategy_name, trade_signal)
    
    if allowed:
        # 执行交易
        execute_order(trade_signal)
        
        # 更新状态
        coordinator.update_positions(strategy_name, {
            "total_value": 计算新总值,
            "positions": 更新持仓列表
        })
    else:
        logger.warning(f"交易被拒绝: {reason}")
```

### **强制平仓规则**
```
1. 超跌模型: 13:00前必须全部平仓
2. 主要模型: 
   - 止损触发时立即平仓
   - 止盈达到时部分或全部平仓
   - 市场环境恶化时减仓
```
